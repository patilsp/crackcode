## üéØ Key MongoDB Features

### 1. Basic Operations
Working with documents and collections:
```javascript
// Create a document
db.users.insertOne({
  name: "John Doe",
  email: "john@example.com",
  age: 30,
  address: {
    street: "123 Main St",
    city: "Boston",
    state: "MA"
  },
  hobbies: ["reading", "gaming"]
});

// Find documents
db.users.find({
  age: { $gt: 25 },
  "address.city": "Boston"
});

// Update documents
db.users.updateOne(
  { email: "john@example.com" },
  { $set: { age: 31 } }
);

// Delete documents
db.users.deleteOne({ email: "john@example.com" });
```

### 2. Indexing
Creating and managing indexes:
```javascript
// Create a single field index
db.users.createIndex({ email: 1 });

// Create a compound index

  name: 1,

// Create a text index
db.products.createIndex({ 
  description: "text" 
});

// Create a geospatial index

  location: "2dsphere"

```

### 3. Aggregation Pipeline
Complex data processing:
```javascript
db.orders.aggregate([
  // Match stage
  { $match: { 
    status: "completed",
    date: { $gte: new Date("2023-01-01") }
  }},
  
  // Group stage
  { $group: {
    _id: "$customerId",
    totalAmount: { $sum: "$amount" },
    orderCount: { $sum: 1 }
  }},
  
  // Sort stage
  { $sort: { totalAmount: -1 }},
  
  // Limit stage
  { $limit: 10 }
]);
```

### 5. Transactions
Working with multi-document transactions:
```javascript
const session = await mongoose.startSession();
session.startTransaction();

try {
  // Perform operations
  await Order.create([{
    userId: user._id,
    items: items,
    total: total
  }], { session });

  await User.updateOne(
    { _id: user._id },
    { $inc: { balance: -total } },
    { session }
  );

  await session.commitTransaction();
} catch (error) {
  await session.abortTransaction();
  throw error;
} finally {
  session.endSession();
}
```

**Q: How do you optimize MongoDB performance with indexes?**

**A:** Here's a comprehensive approach to indexing:

```javascript

db.orders.createIndex({ 
  customerId: 1, 
  orderDate: -1 
});

// Use covered queries
db.users.find(
  { age: { $gt: 25 } },
  { name: 1, age: 1, _id: 0 }

// Monitor index usage
db.users.aggregate([
  { $indexStats: {} }
]);
```

Key points:
- Create indexes based on query patterns
- Use compound indexes for multiple field queries
- Monitor index usage and size
- Use covered queries when possible
- Consider index intersection

### 2. Sharding Strategy
**Q: How do you implement and manage sharding in MongoDB?**

**A:** Here's a sharding implementation:

```javascript
// Enable sharding for a database
sh.enableSharding("myapp");

// Shard a collection
sh.shardCollection(
  "myapp.orders",
  { customerId: 1, orderDate: 1 }
);

// Add shards
sh.addShard("shard1/mongodb1:27017");
sh.addShard("shard2/mongodb2:27017");

// Check sharding status
sh.status();
```

Key considerations:
- Choose appropriate shard key
- Monitor chunk distribution
- Balance data across shards
- Handle shard key updates
- Implement proper backup strategy

### 5. Backup and Recovery
**Q: How do you implement backup and recovery strategies in MongoDB?**

**A:** Here's a backup and recovery approach:

```javascript
// Using mongodump
mongodump --uri="mongodb://localhost:27017/myapp" --out=/backup

// Using mongorestore
mongorestore --uri="mongodb://localhost:27017/myapp" /backup

// Point-in-time recovery
mongodump --uri="mongodb://localhost:27017/myapp" --oplog

// Automated backup script
const { exec } = require('child_process');

function backup() {
  const date = new Date().toISOString().split('T')[0];
  const command = `mongodump --uri="${process.env.MONGODB_URI}" --out=/backup/${date}`;
  
  exec(command, (error, stdout, stderr) => {
    if (error) {
      console.error(`Backup failed: ${error}`);
      return;
    }
    console.log(`Backup completed: ${stdout}`);
  });
}
```

Key points:
- Implement regular backups
- Test recovery procedures
- Monitor backup success
- Store backups securely
- Document recovery procedures

```javascript
// Implement caching
const cache = new NodeCache({ stdTTL: 600 });

async function getCachedData(key) {
  const cachedData = cache.get(key);
  if (cachedData) return cachedData;

  const data = await db.collection('data').findOne({ _id: key });
  cache.set(key, data);
  return data;
}

// Monitor performance

db.serverStatus()
```
